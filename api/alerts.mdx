## Generate Alerts

<api method="POST" path="/alerts/generate" auth="bearer">
  Automatically generate alerts based on inventory thresholds. Admin role required.
</api>

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication (admin role required)
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Indicates if alerts were generated successfully
</ResponseField>

<ResponseField name="alertsGenerated" type="number">
  Number of new alerts created
</ResponseField>

<ResponseField name="message" type="string">
  Success or error message
</ResponseField>

### Example Request

```bash
curl -X POST https://cemac-api.vercel.app/alerts/generate \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json"
```

### Example Response

```json
{
  "success": true,
  "alertsGenerated": 5,
  "message": "Alertas generadas exitosamente"
}
```

### Error Codes

- `401` - Unauthorized
- `403` - Forbidden - Admin role required
- `500` - Internal server error

## Get Alerts

<api method="GET" path="/alerts" auth="bearer">
  Retrieve a list of alerts with filtering and pagination options.
</api>

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

### Query Parameters

<ParamField query="status" type="string">
  Filter by alert status ("pendiente", "en_proceso", "atendido")
</ParamField>

<ParamField query="priority" type="string">
  Filter by priority level ("critica", "urgente", "media", "baja")
</ParamField>

<ParamField query="startDate" type="string">
  Filter alerts from this date (ISO 8601 format)
</ParamField>

<ParamField query="endDate" type="string">
  Filter alerts until this date (ISO 8601 format)
</ParamField>

<ParamField query="page" type="number" default="1">
  Page number for pagination
</ParamField>

<ParamField query="limit" type="number" default="10">
  Number of alerts per page
</ParamField>

<ParamField query="sortBy" type="string">
  Field to sort by (e.g., "createdAt", "priority")
</ParamField>

<ParamField query="sortOrder" type="string">
  Sort order: "asc" or "desc"
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="data" type="object">
  Response data object
</ResponseField>

<ResponseField name="data.alerts" type="array">
  Array of alert objects
</ResponseField>

<ResponseField name="data.pagination" type="object">
  Pagination information
</ResponseField>

### Example Request

```bash
curl -X GET "https://cemac-api.vercel.app/alerts?status=pendiente&priority=urgente&page=1&limit=10" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json"
```

### Example Response

</api>

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

### Path Parameters

<ParamField path="alertId" type="string" required>
  ID of the alert to update
</ParamField>

### Request Body

<ParamField body="status" type="string" required>
  New status: "pendiente", "en_proceso", or "atendido"
</ParamField>

<ParamField body="notes" type="string">
  Optional notes about the status change
</ParamField>

<ResponseField name="success" type="boolean">
  Indicates if the update was successful
</ResponseField>

<ResponseField name="alert" type="object">

</ResponseField>

```bash
curl -X PUT https://cemac-api.vercel.app/alerts/alert_001/status \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "en_proceso",
    "notes": "Pedido realizado al proveedor"
```

### Example Response

```json
{
  "success": true,
  "alert": {
    "id": "alert_001",
    "status": "en_proceso",
    "notes": "Pedido realizado al proveedor",
    "updatedAt": "2025-11-16T11:00:00.000Z"
  }
```
